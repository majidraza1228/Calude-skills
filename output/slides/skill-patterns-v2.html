7 Patterns for Skills That Actually Work

Built from 75+ tests. Every pattern solves a specific failure mode.

Why skills fail

Three problems. The same three. Every time.

01

Wrong skill loads

Short descriptions don't match natural language. Exclusions buried in the body fire too late — the wrong skill is already running.

02

Output drifts

No template, no worked example. Same skill, same prompt — Monday it's markdown headers, Wednesday it's a bullet list. Automation breaks.

03

Bottom half ignored

Safety rules on line 700 of an 800-line skill never fire. Attention degrades. Critical rules at the top, everything else after.

Patterns 1 & 2

The description is a routing table, not a label.

A 54-char description matches almost nothing. A 280-char description with trigger phrases and exclusion redirects routes correctly every time.

✓Target 150–300 chars with 3–5 alternate trigger phrases
✓Every "Do NOT use for X" must include "use /Y instead"
✓Exclusions in the body fire at execution time — already too late

Pattern 3

Claude mirrors the tone of your instructions.

❌ Hedging instructions
Could you take a look and
maybe check if there are issues?
It would be nice to point out
anything if you get a chance.
Output: "You might want to consider renaming this variable — it could be a bit clearer."
✓ Direct imperatives
Flag every issue.
Classify: CRITICAL / WARNING / NIT.
Reference exact file:line.
Output a table. No softening.
Output: "CRITICAL | auth.py:42 | SQL injection | Use parameterized query"

Replace: maybe → Flag · could → Classify · consider → Output · if you get a chance → (cut)

Pattern 4

"Check the relevant files" is not an instruction.

Source Path What to extract
App config/config/app.yamlapp.name, app.version, app.env
DB settings/config/database.ymlhost, port, pool_size
Feature flags/config/features.jsonAll keys where value is true
Deploy log/var/log/last_deploy.logTimestamp of last line, exit code
Env varsprocess.env / .env.localDATABASE_URL, REDIS_URL, LOG_LEVEL

Every "check", "look at", or "review" instruction needs a Source → Path → What to extract table. Vague = wrong data or hallucinated data.
